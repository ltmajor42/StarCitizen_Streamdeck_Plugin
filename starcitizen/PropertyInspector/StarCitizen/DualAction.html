How it works
Overview
  • The Dual Action provides separate behaviors for the button press (down) and release (up) events so you can map two different Star Citizen functions to a single key.
Images / states
  • State 0 — idle/default image shown when the key is not pressed.
  • State 1 — active/pressed image shown while the key is held down (visual feedback for the press action).
Behavior details
  • Press action (Down) — when you press the Stream Deck key the configured "Press action" function is sent as a key down (or mouse down) and the plugin sets the button to State 1. This allows actions that require holding the input.
  • Release action (Up) — when you release the Stream Deck key the plugin sends the corresponding key up for the press action and then triggers the configured "Release action" as a short one-shot (down/up) if defined.
  • Both press and release functions use the plugin's key/mouse/joystick conversion. If a function has no valid input token it will be skipped.
Example
  • Configure Press action = "FireDown" and Release action = "FireUp" → pressing the key sends FireDown (held) and releasing sends FireUp (and returns the image to idle).
  • Or configure Press action = "ThrottleHold" and Release action = "ThrottleTap" to combine a hold-and-tap behavior in one physical key.
Tip: use Dual Action for behaviors that naturally split into a hold while pressed and a separate action on release (e.g., charge/spool, temporary modes).
