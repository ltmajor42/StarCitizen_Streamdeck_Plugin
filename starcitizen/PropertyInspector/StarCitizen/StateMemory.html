How it works
Overview
  • State Memory lets a key remember a binary state (on/off) and optionally send the associated function when toggled. It supports a short-press to fire+toggle and a long-press to toggle memory only (soft sync).
Images / states
  • State 0 — idle/default image when memory is off.
  • State 1 — active image shown when memory is on (visual indicator of stored state or recent activity).
Timing parameters
  • Long press (ms) — threshold to distinguish a short tap from a long press. A press longer than this value triggers the long-press behavior (memory-only toggle).
  • Keypress delay (ms) — delay used when sending the short press key event (down/up) to ensure the target application registers the press.
Behavior details
  • Short press — sends the configured function to the game (a short keypress) and then flips the stored memory state. Use this for toggles that should both act and remember.
  • Long press — when the press exceeds the Long press threshold the plugin does NOT send the function; it only updates the memory state locally (Soft Sync). This is useful to change stored state without sending in-game actions.
  • Soft Sync — toggling memory with a long press updates the plugin's local state without sending the binding to the game. Use Soft Sync for UI-only flips or to correct memory without affecting gameplay.
  • The plugin provides visual feedback by switching between State 0 and State 1; the memory value persists until changed by a subsequent action.
Example
  • Short press: press the key quickly → plugin sends the configured function (short keypress) and toggles memory from off to on; button shows State 1.
  • Long press: hold the key beyond the Long press threshold → plugin updates memory only (Soft Sync) and does not send the function; useful for preparing state without firing actions.
Tip: choose a Long press threshold that is noticeably longer than your normal tap to avoid accidental soft-syncs. Use Keypress delay to improve reliability with games that miss very short pulses.
